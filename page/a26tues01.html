悟以往之不谏，知来者之可追

发表于2022-04-26|更新于2022-04-26|Hexo
|字数总计:1.3k|阅读时长:5分钟

时隔5年，又要从这里开始了。还记得第一次接触到“博客”这个名词还是在大一，在一次技术沙龙中无意中看到知识产出还可以这么优雅，于是乎，开始了 markdown、hexo。自此开始学会用文字记录技术成长。

还记得第一次使用 Hexo 的蹩脚，光一个 nexT 主题就配置了好久，当时是个啥都不知道的小白，不知道什么是css，什么是js（ 因为当时只学过c😁 ）。但是很享受整个折腾的过程，以及成功之后的兴奋。就这样，开始用博客来记录每次遇到的技术难题，虽然现在回过头去看，当前记录的问题有些幼稚，但是，看着自己写的东西有种莫名的亲切，即使是那么幼稚。

自大学毕业之后，能静下心来写作的时日寥寥无几。也有那么一瞬间，我想把工作中遇到的问题，以及解决思路用文字记录下来，但是每次打开typora都不知到从何下笔，几乎每次都是写了一半，不知道自己在干什么，然后delete摁到底，极其烦躁。这也是工作到今天，没有写成过一篇像样的文章。

疫情之下，有了更多的独处时间，工作也不像之前那么忙碌。每当夜深，细数自己的过往，才发现，每天忙忙碌碌，却什么也没留下，甚至回忆不起来一个让我难忘的瞬间。也就是这两天，大学同学来电，聊聊过往，谈谈现在，忽然想起来，还有Hexo，这也是时隔这么长时间，第一次下定决心要重新来过。

Hexo 安装

环境

  • 网络环境：一个好的网络环境，真的可以提升幸福度
  • node.js：Hexo 是基于 开发的静态博客生成工具
  • git：自己的代码管理以及一些工具的下载，要熟悉基本git操作
  • 两个仓库
    • blog空仓库：用于存放 hexo 以及博客文章 markdown 源码
    • your.github.io仓库：用于托管静态页面，详细参照 github pages
